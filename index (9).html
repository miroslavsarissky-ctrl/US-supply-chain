Map tiles can't load from file://

The vendor markers are positioned correctly, but browsers block external map tiles when the page is opened directly from disk. Two ways to fix this:

  1. Serve locally — open Terminal, then run: 
    cd ~/Downloads
python3 -m http.server 8000
    Then open http://localhost:8000/ and click this file.
  2. Deploy to GitHub Pages — drop this file into the US-advanced-reactor-landscape repo as index.html. Tiles will load normally once it's hosted.

All vendor data, filters, and search work without tiles — you can still use the dashboard, just without the basemap.
